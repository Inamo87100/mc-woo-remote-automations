# Support Channels

Where and how to get support for MC-Woo Remote Automations and MC Remote API.

---

## WordPress.org Support Forum (Primary)

**URL**: https://wordpress.org/support/plugin/mc-woo-remote-automations/

This is the **official** support channel for:
- Installation and configuration questions
- Bug reports (non-security)
- Feature requests
- Compatibility questions

**Response time**: We aim to respond within 7 days.

---

## GitHub Issues

**URL**: https://github.com/Inamo87100/mc-woo-remote-automations/issues

Use GitHub Issues for:
- Confirmed bugs (with reproduction steps)
- Feature requests
- Developer discussions

Use the provided issue templates when opening a new issue.

**Response time**: 5 business days for new issues.

---

## Security Vulnerabilities

**DO NOT** report security vulnerabilities in public forums or GitHub issues.

Report security issues privately to: **security@mambacoding.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if known)

**Response time**: Within 48 hours. We follow responsible disclosure (30-day embargo before public disclosure).

---

## Bug Reporting Process

1. Search existing issues first — your bug may already be reported.
2. Use the [Bug Report template](.github/ISSUE_TEMPLATE/bug_report.md).
3. Include:
   - WordPress version
   - WooCommerce version
   - PHP version
   - Plugin version
   - Steps to reproduce
   - Expected vs actual behavior
   - Any error messages from the PHP error log

---

## Feature Request Process

1. Check the [GitHub Issues](https://github.com/Inamo87100/mc-woo-remote-automations/issues) — the feature may already be requested.
2. Use the [Feature Request template](.github/ISSUE_TEMPLATE/feature_request.md).
3. Describe the use case — not just the implementation.
4. Community votes (+1 reactions) help prioritize.

---

## Community Guidelines

- Be respectful and constructive.
- Provide as much context as possible.
- Use English when possible (it helps reach a wider audience).
- No spam, self-promotion, or off-topic posts.
- Do not share credentials, API keys, or sensitive data in public forums.
